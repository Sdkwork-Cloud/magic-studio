# Step 质量审计清单与复盘模板

## 1. 质量审计清单

每个 step 收尾前至少检查以下 10 项：

1. 是否对齐具体架构文档编号
2. 是否明确输入、非目标、输出
3. 是否精确到关键代码目录和高风险文件
4. 是否给出明确检查点
5. 是否给出真实测试计划
6. 是否给出可重复执行的结果验证命令
7. 是否明确串行 / 并行边界
8. 是否明确风险与回滚
9. 是否完成架构回写
10. 是否满足下一步准入条件

## 2. 评分建议

| 维度 | 分值 |
| --- | --- |
| 架构对齐 | `15` |
| 边界清晰 | `10` |
| 实施可执行性 | `20` |
| 测试完整性 | `15` |
| 结果验证 | `10` |
| 并行编排合理性 | `10` |
| 风险与回滚 | `10` |
| 架构回写与交接 | `10` |

建议分数不低于 `90/100`。

## 3. 复盘模板

```md
# Step XX 复盘

## 完成项

## 未完成项

## 检查点通过情况

## 测试与验证结果摘要

## 残余风险

## 回滚是否可执行

## 是否允许进入下一步

## 对 docs/架构 的回写项
```
