# Step 输入输出与阻塞升级规则

## 1. 输入输出规则

每个 step 必须明确：

- 执行输入：允许依赖的代码、文档、脚本、测试基线
- 本步非目标：本 step 不能顺手扩张的范围
- 最小输出：本 step 完成后必须交出的能力和证据

## 2. 阻塞分级

| 等级 | 定义 | 处理方式 |
| --- | --- | --- |
| `B1` | 文档或上下文不清 | 由 owner 当天补齐 |
| `B2` | 跨 lane 接口不一致 | 升级给架构 owner，当天冻结 |
| `B3` | 共享文件冲突 | 停止并行，按 owner 规则串行 |
| `B4` | 测试或构建阻断 | 优先修复，不得跳过 |
| `B5` | 安全或数据破坏风险 | 立即停止实施，走回滚评审 |

## 3. 升级规则

- `B1-B2`：lane 内解决
- `B3`：集成 owner + 架构 owner 决策
- `B4-B5`：必须阻断当前 step 完成声明

## 4. 执行卡最小模板

```md
## 当前输入
## 本步非目标
## 当前最小输出
## 当前阻塞等级
## 需要谁决策
## 当前是否允许继续推进
```
